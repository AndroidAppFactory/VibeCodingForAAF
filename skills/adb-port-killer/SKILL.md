# ADB 端口释放工具

## 工作流程

```
1. 执行 lsof -wni :5037
   ├─ 无输出 → 提示"端口未被占用"，结束
   └─ 有输出 → 按下方格式展示，等待用户选择
       ↓
2. 按用户输入执行对应操作
   - kill -9 失败时报告原因（权限不足、进程已退出等）
       ↓
3. 再次 lsof -wni :5037 验证释放结果
```

## 展示格式

进程列表编号从 1 开始，固定选项紧跟在最后一个编号之后：

```markdown
以下进程正在占用 ADB 端口 (5037)：

| # | PID | 进程名 | 用户 | 连接状态 |
|---|-----|--------|------|----------|
| 1 | 1234 | adb | zixie | LISTEN |
| 2 | 5678 | Python | zixie | ESTABLISHED |

请选择操作：
- 输入编号终止对应进程（多个用空格分隔，如 `1 2`）
- 3: 尝试 adb kill-server 优雅关闭
- 4: 取消
```

> 编号动态生成：N 个进程编号 1~N，`adb kill-server` = N+1，取消 = N+2。
